## Implementation Notes

### Survey URL Parameters
All templates use the same URL structure:
```
https://your-opencsat-domain.com/api/survey/create-and-redirect?
  ticket_id=VALUE&
  customer_email=VALUE&
  customer_name=VALUE&
  ticket_subject=VALUE&
  technician_name=VALUE&
  company_name=VALUE&
  completion_date=VALUE&
  [additional_custom_fields...]
```

### Setup Instructions for MSPs

1. **Replace Domain**: Change `your-opencsat-domain.com` to your actual OpenCSAT domain
2. **Test Templates**: Use test tickets to verify merge fields work correctly
3. **Customize Branding**: Update colors, logos, and company information
4. **Configure Triggers**: Set up workflow rules to send emails when tickets are closed/resolved
5. **Monitor Results**: Check OpenCSAT dashboard for survey responses

### Benefits of This Approach

- **Universal Compatibility**: Works with any PSA that supports email templates
- **No Complex Integrations**: Just modify existing email templates
- **Rich Ticket Context**: Surveys show full ticket details for better feedback
- **Immediate Deployment**: Start collecting feedback without API setup
- **Scalable**: Add new PSA systems easily with template modifications
